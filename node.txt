But problem was not in language (of course you can use cache and LB, but...).

Problem was, that Rails puts a global lock for incoming request, and doing nothing
with other requests - so it was a single threaded program.

People was uploading files to server.

And uploading big files with POST was pain - all freezes.

Second way - long pooling.

**Idea of long pooling**

So Ryan whants a server optimized for this behaviour.

Итак Ruby был заброшен. Была попытка написать весь веб сервер на C. Но веб
